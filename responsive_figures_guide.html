Responsive Figure Layout Examples

Demonstration of how to layout multi-subplot figures responsively using separate SVG files.

Pattern 1: CSS Grid (Auto-responsive)

Best for: Equal-sized subplots that should wrap naturally

Housing Budget
Households by Burden
Affordability Gap
<div class="figure-grid"> <div><img src="subplot1.svg"></div> <div><img src="subplot2.svg"></div> <div><img src="subplot3.svg"></div> </div> .figure-grid { display: grid; grid-template-columns: repeat(auto-fit, minmax(300px, 1fr)); gap: 20px; }

Pattern 2: Flexbox (Wrapping)

Best for: Varying subplot sizes that should wrap

Housing Budget
Households by Burden
Affordability Gap
<div class="figure-flex"> <div class="figure-flex-item"><img src="subplot1.svg"></div> <div class="figure-flex-item"><img src="subplot2.svg"></div> <div class="figure-flex-item"><img src="subplot3.svg"></div> </div> .figure-flex { display: flex; flex-wrap: wrap; gap: 20px; } .figure-flex-item { flex: 1 1 300px; min-width: 300px; }

Pattern 3: Responsive Grid with Breakpoints

Best for: Custom layouts at different screen sizes (most control)

Housing Budget
Households by Burden
Affordability Gap
/* Mobile: Stack */ @media (max-width: 768px) { .figure-responsive { grid-template-columns: 1fr; grid-template-areas: "left" "top-right" "bottom-right"; } } /* Desktop: Custom layout */ @media (min-width: 1025px) { .figure-responsive { grid-template-columns: 1fr 1fr 1fr; grid-template-areas: "left top-right top-right" "left bottom-right bottom-right"; } }

Pattern 4: Container Queries (Most Modern)

Best for: When layout should adapt to container size, not viewport

Note: Requires modern browsers (2023+)

Housing Budget
Households by Burden
Affordability Gap
.figure-container { container-type: inline-size; } @container (max-width: 600px) { .figure-container-grid { grid-template-columns: 1fr; } } @container (min-width: 901px) { .figure-container-grid { grid-template-columns: repeat(3, 1fr); } }

Recommendation for Your Website

Use Pattern 3 (Responsive Grid with Breakpoints)

Why:

  • ✅ Full control over layout at each screen size
  • ✅ Works with your existing fixed-image scroll design
  • ✅ Can match your specific design needs (e.g., donut on left, charts stacked on right)
  • ✅ Excellent browser support
  • ✅ Easy to integrate with your current CSS
